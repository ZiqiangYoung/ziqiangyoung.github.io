OpenCV笔记

更新于 2021-05-05 阅读次数： 本文字数： 4.7k 阅读时长 ≈ 4 分钟

最近为了满足一个数字图像处理需求,稍微入门了一下openCv,特此整理用到的主要内容。

图片操作

读取图片

  • 宏cv2.IMREAD_COLOR指读取模式，COLOR–>BRG，GRAYSCALE–>GRAT

  • 需要注意的是imread读取出来的并非Image对象，故此不要随意和IMAGE model混用

  • 该方法默认以BGR通道读取图像，不同于Image.open的RBG读取方式

    • 可以用cvtclolr将BGR转RGB，一般用于与matplotlib配合使用。
    • cvtcolor还可用于将图片灰度化 。

图像显示

openCV

非常不好用，显示小图还可以，大图显示不全，默认不能缩放且缩放非常烂。

  • winName为显示图像的窗口名称
  • image为需要显示的图像
  • waitkey≈system pause 但光标在图像窗口输入才有效

matplotlib.pyplot

图像预览比较方便，缩放功能比较好使

Tips:pyplot使用时注意通道顺序,imread以BGR读取图像，而pyplot默认RGB

图像保存

imread默认按照BGR读取的图片，与imwrite配合通道不会搞混通道

  • filePath保存的文件路径
  • image需要保存的图片

图像截取

X对应于图片横坐标，Y对应于图片纵坐标，当成矩阵对待。

缩放图片

  • targetX对应你希望缩放到的大小的横轴值

  • targetY对应你希望缩放到的大小的纵轴值

Tips:我只需要应用缩小图片，放大图片涉及到插值算法等方面，我并未深入

获取尺寸

  • shape为(row,cols,channel)分别为:矩阵行、矩阵列、通道数
  • :2表示except[2]，即除了第三个元素，是对元组的简单截取。

绘制矩形

在图片中的某个位置绘制矩形/矩形框

  • outputImage是需要绘制矩形/矩形框的图片
  • top_left指需要绘制的图片的左上角坐标
  • bottom_right为所需矩阵的右下角坐标
  • 根据这两点坐标，即可确定矩阵位置
  • channel_value指你需要绘制的矩形的通道值，例如对于灰度 255 对于BGR (0,255,0)
  • para指绘制参数，-1则填充整个矩形，1则只绘制细矩形框，2表示绘制粗矩形框。

阈值分割

这一过程也叫做图片的二值化。

我只应用了简单阈值，并未详细深入

参考：

中文–图像阈值

  • retpython多值返回中的一个值，忽略掉即可
  • img需要阈值分割的图片
  • threshold阈值，0-255之间的某个值
  • value阈值分割的结果值，比如需要分割为0和255，则value为255

简单阈值

通过设置一个阈值，将单通道图片中低于阈值的像素置为0，高于阈值的像素置为255。

这种方式只能处理极其简单的图片，没有光线等因素的影响

一般用于处理灰度图像(因为该操作是对于单通道的)

形态学处理

中文–形态学操作
英文–Morphological Transformations

定义卷积核

腐蚀

腐蚀的效果是把目标”变瘦”，其原理是在原图的卷积核小区域内取局部最小值。

特别对于二值化图来说，小区域内有一个是0则该坐标的卷积结果就为0。

腐蚀会消除白噪声，但也会缩小物体。

  • 迭代次数iterations代表连续腐蚀多少次。

膨胀

膨胀与腐蚀相反，取的是局部最大值，效果是把目标”变胖”

膨胀会增加图像中的白色区域或增加前景对象的大小。

  • 迭代次数iterations代表连续膨胀多少次。

开运算

开运算只是先侵蚀再扩张的另一个名称。

通常在消除噪点时选择先腐蚀再膨胀，因为腐蚀会消除白噪声，但也会缩小物体，因此再对其进行膨胀。

效果就是噪点消失，待检测目标大小基本正常，还可以将”粘连”的多个物体分开。

闭运算

闭运算为先扩张再侵蚀，与开运算相反。

闭运算在消除目标上的黑噪点时有效。

轮廓寻找

参考：

中文–轮廓入门

中文–轮廓

轮廓是图片中物体的边缘点的集合。

图片经过形态学处理(提高结果准确度)后二值化(只能对二值化图像操作)，就可以进行轮廓寻找

  • contours是寻找出来的轮廓的集合，一维数组，0-len 是由最外层到最内层的轮廓编号
  • hierarchy是轮廓的树形结构，包含轮廓之间的父子兄弟关系
  • thresh是你需要进行轮廓查找的二值化图片
  • cv2.CHAIN_APPROX_SIMPLE是轮廓近似方法，该方法对轮廓以线段方式存储来节省空间
  • cv2.RETR_TREE表示查找到轮廓同时返回亲属树

绘制轮廓

  • img是需要绘制轮廓的图片
  • channel_value指你需要绘制的矩形的通道值，例如对于灰度 255 对于BGR (0,255,0)
  • which指需要绘制的轮廓的编号,-1为全部绘制
  • para指绘制参数，-1则填充整个矩形，1则只绘制细矩形框，2表示绘制粗矩形框

模板匹配

模板匹配是指给定一张图片(模板)然后在另一幅图像中寻找这个模板的操作。它是一种用来在一幅大图中寻找模板图像位置的方法,若两图片相同或者大小逆置，返回-1

  • 它的工作原理为卷积

  • 可视为滑动平均推广， 将模板图像在输入图像(大图)上滑动，在每一个位置进行比较

  • result是一个单通道图像，每一个像素值表示了此区域与模板的匹配程度

  • 宏cv2.TM_CCOEFF_NORMED代表希望选择的匹配算法

匹配步骤

  1. 输入原图像(I)和模板图像(T)。在原图像中我们希望找到一块和模板匹配的区域

  2. 通过将模板在原图像上滑动来寻找最匹配的区域。 这里所谓的滑动是指模板图像块一次移动一个像素(从左往右，从上往下)。 在每一个位置，都进行一次度量计算，来判断该像素对应的原图像的特定区域 与模板图像的相似度。

  3. 对于模板T覆盖在I上的每个位置，把上一步计算的度量值保存在结果图像矩阵R中。 在R中每个位置都包含对应的匹配度量值。

  4. 在结果图像矩阵中寻找最值(最大或最小，根据算法不同而不同)。最值所对应的像素的位置即认为是最高的匹配。 以该点为顶点，长宽和模板大小图像一样的矩阵认为是匹配区域。

匹配算法

在匹配步骤中有个核心问题， 那就是如何度量匹配程度，如何进行度量计算。

OpenCV提供了6种不同的基础匹配方法。

在下列公式中， T(x,y)表示模板，I(x,y)表示原图像，R(x,y)表示模板与原图相似度函数。 x、y表示某个像素，x’、y’表示循环变量，在模板那么大的范围内进行循环滑动。

平方差匹配(TM_SQDIFF)

将模板中的像素依次与对应原图中的区域作差再平方求和，得到R。匹配越好，差值越小，R值越小。

标准平方差匹配(TM_SQDIFF_NORMED)

这个公式其实是对图像和模板进行了标准化操作，这样可以保证当模板和图像各个像素的亮度都乘上同一个系数时，相关度不变。 即当I(x,y)和T(x,y)变成kI(x,y)和kT(x,y)时，相关系数还是R(x,y)。与上面一样，匹配越好，R值越小。

相关匹配(TM_CCORR)


这类方法采用模板和图像间的乘法操作，所以较大的数表示匹配程度较高，0标识最坏的匹配效果。

标准相关匹配(TM_CCORR_NORMED)


这种方法和第二种类似，同样是通过标准化从而去除了亮度线性变化对相似度计算的影响。匹配越好R值越高。

相关匹配(TM_CCOEFF)

虽然也叫做相关匹配，但是和上面的还是有点不同。不同之处在于，把原图和模板都减去了各自的平均值。

。

标准相关匹配(TM_CCOEFF_NORMED)

相比于上一个方法，这里又除了各自方差。经过减去平均值和除以方差这么两步操作之后， 无论是我们的待检图像还是模板都被标准化了， 这样可以保证图像和模板分别改变光照亮不影响计算结果。

获取坐标

前文提起过result是一副单通道图像，每一个位置都代表了模板在原图滑动时对应位置算出的相关系数。

所以只需要找出最匹配的那一点的坐标即可。

在OpenCV中可以用cv2.minMaxLoc()函数获得单通道图片的通道最值坐标。

  • minValue指系数最小的点的系数值，maxValue同理
  • minValue location指系数最小的点的坐标，maxValue location同理
  • 相关系数与最佳匹配之间的关系对于不同的匹配算法不尽相同
  • 根据TM_CCOEFF_NORMED算法可知，完全匹配的值为1，即bestMatch = maxValue

轮廓识别是用直线来围成轮廓，而不是用点来围成轮廓，节省了空间

轮廓识别(生成关系树)对最外层是取内侧像素点，对下一层是取外侧像素点

奇数轮廓取外侧像素点，偶数轮廓取内侧像素点，像素级运算求长度时应当对齐,内侧+1，外侧-1

求两个轮廓（偶数轮廓-奇树轮廓）之间的距离，用leftmost1-leftmost2-1
